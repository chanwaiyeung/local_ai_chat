# RAG Stability And Quality Checklist

Use this checklist after changing embedding models, vector store behavior, RAG retrieval, citation links, or document ingestion.

## 1. Setup

- Run the app in debug mode:

```powershell
cd C:\dev\local_ai_chat
& "C:\src\flutter\flutter\bin\flutter.bat" run -d windows
```

- Keep the terminal visible and search for:

```text
RAG ingest
RAG retrieve
SettingsScreen
```

- When testing the release exe directly, open the app menu and choose
  `RAG debug log 位置`, then inspect `rag_debug.log`. Release builds may not
  show `debugPrint` output in a terminal, so this file is the source of truth
  for release-candidate RAG logs.

- Prepare at least:
  - one small PDF/TXT/MD
  - one larger PDF, ideally over 5 MB
  - 5 factual questions whose answers exist in the file
  - 3 questions whose answers do not exist in the file
  - 3 synonym/paraphrase questions
  - 2 follow-up questions

- Confirm the embedding model is installed before applying it:

```powershell
ollama list
ollama pull nomic-embed-text
ollama pull bge-m3
```

If the model is not installed, the app should reject the change and show:

```text
Embedding model 未安裝：<model>
```

## 2. Embedding Model Switch Stability

Run this loop 3 to 5 times:

1. Open `Embedding 設定`.
2. Switch between `nomic-embed-text` and `bge-m3`.
3. Confirm `清空並套用`.
4. Upload the same test document.
5. Confirm the assistant message includes:

```text
Embedding model：<selected model>
```

6. Confirm debug log includes:

```text
Embedding changed: old=... new=... storeCleared=true storeChunks=0
RAG ingest: start doc=... embeddingModel=<selected model>
RAG ingest: done doc=... embeddingModel=<selected model> chunks=... durationMs=... storeEmbeddingModel=<selected model>
```

Pass criteria:

- No crash.
- No stale model name in logs.
- Vector store chunk count matches the newly ingested document.
- Vector store embedding metadata matches the current embedding model.
- RAG banner shows `Top-K · <selected model>`.

## 3. Duplicate Ingest

1. Upload the same document twice with the same embedding model.
2. Open the document library.
3. Check chunk count for that document.

Pass criteria:

- Chunk count does not double unexpectedly.
- The latest upload replaces the previous chunks for the same document.
- Debug log shows one final store chunk count consistent with the document.

## 4. Large File Ingest

1. Upload a PDF over 5 MB.
2. Watch for progress and completion.
3. Ask one simple factual question from the file.

Record:

Pass criteria:

- UI remains responsive enough to recover after ingest.
- No failed writes to vector store.
- A reasonable citation appears for factual questions.

## 6. Debug Log Interpretation

Expected ingest logs:

```text
RAG ingest: start doc=report.pdf embeddingModel=bge-m3 chars=12345
RAG ingest: done doc=report.pdf embeddingModel=bge-m3 chunks=18 durationMs=1234 storeChunks=18 storeEmbeddingModel=bge-m3
```

Expected mismatch safeguard log:

```text
Embedding mismatch detected: store=nomic-embed-text current=bge-m3 storeCleared=true
```

Expected retrieve logs:

```text
RAG retrieve: start query="..." embeddingModel=bge-m3 doc=report.pdf topK=4 chunks=18
RAG retrieve: done hits=4 durationMs=120 scores=[report.pdf#3:0.842, report.pdf#7:0.801]
```

If retrieval quality is poor:

- Compare scores between `nomic-embed-text` and `bge-m3`.
- Check whether the cited chunk actually contains the answer.
- Check whether grounding rejected useful results because wording differs.
- Add the case to RAG Evaluation Panel with expected status and verdict.

## 7. Regression Smoke Test

After the RAG checks, confirm:

- Chat model dropdown still changes chat model only.
- Embedding setting changes embedding model only.
- RAG Evaluation Panel records the selected embedding model.
- Markdown code blocks still render and copy.
- Citation links still open DocViewer and highlight chunks.
- External links still open via browser/mail app.
