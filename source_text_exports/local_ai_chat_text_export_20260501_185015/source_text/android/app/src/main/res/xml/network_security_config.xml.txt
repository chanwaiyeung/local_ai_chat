<!--
  Network Security Config for the AI Library client.

  Why we permit cleartext: the embedded server runs on the user's own
  desktop, on a LAN address (10.0.2.2 from the Android emulator, or a
  192.168/10/172.16 address from a real device). HTTPS would require a
  self-signed cert and on-device trust setup that's hostile to the
  "drop a folder, run one command" UX this product is built around.

  Why we don't try to scope this with <domain> rules: Android's
  <domain> element only accepts exact hostnames or `*.suffix.example`
  via includeSubdomains. It does NOT support IP prefixes like
  `192.168.` — such an entry silently matches nothing. So the only
  honest options are blanket cleartext (this file) or full TLS
  (deferred). We pick the first.

  base-config cleartextTrafficPermitted="true" is functionally the
  same as the legacy `android:usesCleartextTraffic="true"` attribute
  on <application>, but Android prefers configs over attributes from
  API 24+ and surfaces this in lint output.
-->
<network-security-config>
    <base-config cleartextTrafficPermitted="true">
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </base-config>
</network-security-config>
