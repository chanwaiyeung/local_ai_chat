## Current Architecture Locks

- Server is loopback-only by default: `127.0.0.1`.
- LAN mode is explicit and requires a non-empty token.
- Retrieve-first API is available:
  - `GET /docs/<docName>/chunks`
  - `POST /rag/retrieve`
- Existing LLM streaming API remains available:
  - `POST /query`
  - `POST /query/stream`
- Reader UI routes through `ReaderController`.
- `ReaderController.askQuestion()` streaming behavior must remain intact.
- `ReaderController.loadDocument()` and `ReaderController.search()` are
  retrieve-first paths and must not call the LLM.

## Persistence Note

This project uses `data/vectors.ndjson` with an in-memory cosine/MMR
`VectorStore`. It is not the older `local_ai_chat` v2.0 schema-v3 persisted
BM25 store. Treat migration from v2.0 as a separate task.

## Packaging Rules

When creating ZIPs for another AI:

- Include source, tests, scripts, docs, seed books, and `data/vectors.ndjson`.
- Exclude `.dart_tool/`, `build/`, package output folders, `_manus_drop/`,
  and `windows/flutter/ephemeral/`.
- Keep platform scaffolding (`android/`, `web/`, `windows/`) because the app
  must still build from a clean checkout.
