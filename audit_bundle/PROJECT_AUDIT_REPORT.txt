1) WHAT THIS REPOSITORY IS
------------------------------------------------------------------------------
Single Flutter app "local_ai_chat" (智讀館 / AI Library) version ~2.5.x.

It combines:
  A) LIFE / PERSONAL HUB — expenses, contacts, health, wealth, books,
     personal RAG query, skills, reader, library, chat, settings, Telegram bot.
  B) CHURCH MODULE — under lib/screens/church, lib/controllers/church,
     lib/models/church, lib/widgets/church (person directory, care cases,
     visit logs, dashboards).

Entry point: lib/main.dart
  - Initializes VectorStore, controllers (expense, contact, health, wealth,
    book, church care & person), PersonalRagService, SkillsService, optional
    Telegram bot, desktop ApiServer, CurrencyService.load().
  - home: PersonalHubScreen (wires bookController + personalRagService).

Data pattern: most domain objects persist via VectorStore + DocChunk metadata
(Pattern A, same idea as books/wealth).
